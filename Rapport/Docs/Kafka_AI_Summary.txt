SaveSafe-Wallet - Resume Kafka + AI

Objectif
- Centraliser les logs de login et les tentatives de connexion dans Kafka.
- Ajouter un service AI dedie pour analyser ces evenements.
- Produire un score de risque exploitable par le dashboard admin.

Ce qui a ete mis en place
1. Kafka
- Kafka et Zookeeper sont declares dans docker-compose.
- Les services .NET ont maintenant une configuration Kafka.
- Un projet partage a ete ajoute pour factoriser la production et la consommation des messages.
- Un contrat de message LoginEventMessage a ete cree pour standardiser les logs de connexion.

2. AuthService
- AuthService envoie maintenant les evenements de login dans Kafka.
- Les tentatives reussies et echouees peuvent etre publiees de maniere asynchrone.
- Le service n'est plus oblige de traiter toute la logique d'analyse directement.

3. AI service
- Un service ai-security-service a ete cree.
- Il consomme les messages login-events depuis Kafka.
- Il prepare l'analyse du risque a partir des donnees de connexion.
- Pour l'instant, la logique AI est un moteur simple de scoring, pas encore un vrai modele entraine.

4. Dashboard admin
- Le but est d'afficher les resultats de l'analyse AI dans le dashboard admin.
- Le dashboard pourra afficher le score de risque, l'historique des connexions suspectes et les alertes.

Ce que cela apporte
- Architecture plus propre avec event-driven flow.
- Separation entre authentification, analyse et affichage.
- Base technique prete pour brancher un vrai modele de machine learning plus tard.

Etat actuel
- La partie infrastructure Kafka et le service AI existent.
- Le vrai modele AI n'est pas encore entraine.
- La compilation doit encore etre validee sur certaines classes partagées.
