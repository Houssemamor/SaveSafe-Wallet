SaveSafe-Wallet - PASSATION TECHNIQUE
Integration AI dans le Dashboard Admin

OBJECTIF DU TRAVAIL
Afficher dans le dashboard admin les resultats de l'analyse des tentatives de login afin de donner a l'administrateur une vue immediate du risque de securite.

CE QUI A ETE AJOUTE

1. Cote backend .NET
- Un score de risque admin a ete ajoute au resume de securite.
- Le score est calcule a partir des signaux existants:
  - nombre d'echecs de connexion
  - nombre d'evenements flagges
  - nombre d'adresses IP distinctes
  - bonus si beaucoup de connexions sont detectees sur une courte periode
- Un niveau de risque est derive automatiquement: Low, Medium, High.
- Les donnees sont persistees dans Firestore dans la collection admin_stats.
- L'API /api/admin/security-summary expose maintenant les champs AI.
- L'endpoint /api/admin/security-summary/refresh permet de recalculer le resume a la demande.

2. Cote frontend Angular
- Une carte AI Risk Score est affichee dans le dashboard admin.
- La couleur change selon le niveau de risque.
- La date de calcul est affichee.
- Une colonne AI Flag a ete ajoutee dans la liste des evenements de connexion.

3. Infrastructure evenementielle
- Kafka est utilise pour publier les login events.
- Un consumer persiste les evenements.
- Le service ai-security-service est prepare pour consommer les events et produire une analyse.

COMMENT TESTER
1. Lancer: docker compose up -d --build
2. Ouvrir le dashboard frontend sur http://localhost:4200
3. Se connecter avec un compte admin
4. Verifier la carte AI Risk Score et la colonne AI Flag
5. Faire quelques tentatives de login echouees pour voir le score evoluer
